## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it by:

1. Opening a private GitHub security advisory (if available)
2. Or contacting: SECURITY_CONTACT_HERE

**Important:** Please do not disclose the vulnerability publicly until we have had a chance to respond and coordinate a fix. Public disclosure before coordination may be considered a security incident.

## Expected Response Time

We aim to respond to all security reports within **[X] business hours**. Critical vulnerabilities will be prioritized for immediate attention.

## What to Include in Your Report

When reporting a vulnerability, please include:

- A clear description of the vulnerability
- Steps to reproduce or demonstrate the issue
- Any relevant logs, screenshots, or error messages
- The version of the software affected
- Your contact information (optional, for follow-up)

## Do Not Include

- Secrets, API keys, or credentials
- Sensitive production data
- Personally identifiable information (PII)

## Security Best Practices

- Never commit secrets, API keys, or credentials to this repository
- Use environment variables or secrets management for sensitive data
- Keep dependencies updated to the latest secure versions
- Review and follow security best practices for your specific technology stack

## Security Scanning

This repository uses automated security scanning tools to detect vulnerabilities in dependencies and code. Please review the results of these scans and address any reported issues promptly.
